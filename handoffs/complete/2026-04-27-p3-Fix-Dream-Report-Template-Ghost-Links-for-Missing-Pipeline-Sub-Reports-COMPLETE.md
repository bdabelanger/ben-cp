## Context

Every Dream cycle, the links sensor flags 4 ghost links in `reports/dream/report.md`. These are not hand-authored errors — they are generated by the dream report template itself. Two links point to `reports/status/report.md` and `reports/tasks/report.md` (which exist transiently but may not be present when the links sensor runs). Two links evaluate to `None` because the intelligence harvest and intelligence scan pipelines do not write dedicated report files — their output is only captured as inline text in the main dream report.

This will recur on every run until the template is patched.

## Goal

Make the dream report template handle missing or non-existent pipeline sub-report paths gracefully so the links sensor stays clean.

## Execution Steps

1. Locate the dream report generation script — likely in `src/` or `skills/dream/` — and find where pipeline sub-report links are rendered.
2. For each pipeline, check whether a sub-report file actually exists before rendering a markdown link. If the path is `None` or the file doesn't exist, render plain text (e.g., `No report file`) instead of a link.
3. For `reports/status/report.md` and `reports/tasks/report.md`: confirm these are written before the links sensor runs, or make the link conditional on file existence.
4. Re-run the dream cycle and confirm the links sensor returns 0 ghost links.
