# Add Intentional Root Dirs to Drift Sensor Ignore List - dist node_modules src reports

## Context

The drift sensor flagged four root-level directories as `unsanctioned_root_dir`:
- `dist/`
- `node_modules/`
- `src/`
- `reports/`

Ben confirmed all four are intentional. They are not drift — they are legitimate parts of the repo/pipeline structure. The sensor just needs to be told to ignore them.

Ben's preference: each sensor should have an `ignore` config key for paths/patterns to skip during scanning (not "whitelist", "allowlist", or "sanctioned_dirs" — use `ignore` throughout).

## Logic

Add all four directories to the drift sensor's `ignore` list so they no longer trigger WARN findings. Standardize the config key name to `ignore` across all sensors.

## Execution Steps

1. [ ] Locate the drift sensor config — likely in `src/` or `skills/dream/`.
2. [ ] Add or rename the exclusion config key to `ignore`.
3. [ ] Add `dist`, `node_modules`, `src`, and `reports` to the `ignore` list.
4. [ ] Check all other sensor configs (links, frontmatter, agents, pulse, etc.) — rename any equivalent exclusion keys to `ignore` for consistency.
5. [ ] Re-run `generate_report(skill='dream')` and confirm the drift sensor shows 0 findings.
