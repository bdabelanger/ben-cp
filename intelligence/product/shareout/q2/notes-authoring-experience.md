# Slide 16 — Notes Authoring UX: Tabbed Design (Q2)

- **Headline:** Intuitive notes authoring: Reduce clicks and save time.
- **Quote (Path of Life):** "Case managers input bulk notes so that they don't have to write the same thing five to eight times".
- **Tabbed Design:** Clean separation of note narrative, services, and people.
- **Inline Selection:** Add services and people directly within the authoring flow without modal jumps.
- **Service Groups:** Load pre-defined sets of services with one click.
- **"Attended = Yes" Auto-completer:** Rapidly mark roster attendance.
- **Date Handling:** Improved UX for managing note date vs. service interaction dates.
- **Rich Content:** Support for rosters, resource linking, and complex narratives ("Bears! Oh my!").
- **Consistency:** Uses the new tabbed design standard for high-density data entry.

## Strategic Context & Vision
*   **Roadmap Theme:** Compliance and Case Management Roadmap.
*   **The Leap Forward:** This version represents a giant leap for the Casebook Notes experience—comparable to the impact of launching group service notes in 2022. 
*   **Future State:** This sets the stage for Dynamic Pages in Notes, allowing customers to customize and arrange fields later this year.

## The User Experience: Consistency Meets Power
Pierre’s design, implemented by Bisoye, Russell, and Tuan, ensures every note type is consistent and always at eye-level. While the new tabs feel like a big change, they are built upon familiar structures, making them quick to adopt and feeling significantly more mature than their predecessor.

**Customer Voice:** *“[Case Workers] input bulk notes so that they don't have to write the same thing five to eight times.”* – Path of Life (Discovery call)

## Capabilities & Implementation Details
*   **Standardization:** We’ve standardized the look and feel across all note types.
*   **Service Integration:** For Service Notes, we’ve integrated every data entry shortcut built for Services directly into this experience. This now includes **Service Groups**, in addition to Rosters, Bulk actions, and Multiple resource linking.
*   **The Canvas Concept:** We call this the 'canvas' because it points toward our future: Dynamic Pages support, which will allow customers to add and arrange fields themselves later this year.
