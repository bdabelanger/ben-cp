# Styles: Visual Glossary (Emoji Indicators)

This is the canonical reference for all visual status indicators and section markers used across the repo. All reports and documentation must adhere to these standards to ensure cognitive consistency.

## Status Indicators

### ✅ Success & Completion
- **✅ Done/Achieved**: KR has met or exceeded target.
- **👏 Outstanding**: Highest performing entity.

### ⚠️ Caution & Risk
- **⚠️ At Risk/Needs Attention**: Poor trend or internal metric approaching threshold.
- **🟡 Partial/In-Progress**: Significant progress made but not yet complete.
- **🎯 On Track**: Normal execution / monitored.

### ❌ Failure & Blockers
- **🛑 Blocked/Off Track**: External or internal dependency preventing progress.
- **❌ Missed/Failed**: Performance targets definitively not met.

### 👀 Data Quality
- **👀 Missing Info**: Required data fields are absent.

## Directional Icons
- **🔼 Improving**: Trend is moving in a positive direction.
- **🔽 Declining**: Trend is moving in a negative direction.
- **〰️ Stable**: No significant change in trend.

## Formatting Standards
- Use **Bold** for emphasis on critical metrics.
- Use `inline code` for technical paths or IDs.
- Use > [!NOTE] blocks for context that shouldn't clutter the main flow.
