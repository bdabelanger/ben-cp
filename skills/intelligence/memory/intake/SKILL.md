# Skill: Memory Intake

> **Description:** Standardized procedure for capturing significant findings and patterns into persistent memory via `append_note` and structured encoding.
> **Preferred Agent:** Vault Auditor
> **Cadence:** Post-session / Daily Digest completion

## Connections
- **Input:** Ephemeral session notes, dream outputs, and user strategic pivots.
- **Output:** Knowledge Items (KIs), updated domain intelligence, and memory ledger entries.

## Tool Utility
- **mcp_ben-cp_write_intelligence**: Core tool for encoding structured metadata into the knowledge domain.
- **mcp_ben-cp_append_note**: Core tool for updating the memory ledger and notes.

## Workflow Summary
1. **Intake:** Scanning ephemeral sources for candidates of persistent value.
2. **Identification:** Isolating architectural rules, status thresholds, or repeated failures.
3. **Encoding:** Formatting findings into structured domain context (KIs, logic stubs, or mappings).
4. **Finalization:** Updating the memory ledger (changelog.md).

## Constraints
- Focused on "Persistent Value": Only encode information that survives the specific session context.
- Requires signature and ledger recording for every memory ingestion.
