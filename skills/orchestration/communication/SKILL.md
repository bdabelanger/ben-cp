# Skill: Communication

> **Description:** Shared intelligence layer for peer-level collaboration between human user and all agents.
> **Preferred Agent:** Joint Operating Environment (Peer Team)
> **Cadence:** Ad-hoc / Continuous

## Connections
- **Input:** Raw ground truth, ad-hoc decisions, and context from human user and all agents.
- **Output:** Synchronized state and persistent scratchpads across all skill domains.

## Tool Utility
- **replace_file_content**: Used to append signed entries to `notes.md` files without overwriting existing context.
- **grep_search**: Used to scan for recently signed entries or specific user directives across the vault.

## Workflow Summary
1. **Saturation:** Reading all relevant `notes.md` files before planning or execution.
2. **Articulation:** Contributing peer-level observations and status updates to the shared ledger.
3. **Synchronization:** Ensuring all agents and the human user are aligned on blockers and strategic shifts.

## Constraints
- **Peer-Level Tone:** Collaborative, direct, and non-subordinate.
- **Persistence:** Never edit or delete another agent's or human user's entries.
- **Signature Required:** Every entry must be signed and timestamped.
