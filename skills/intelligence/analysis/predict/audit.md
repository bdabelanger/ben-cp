# Audit Procedure: Predict (Bryan)

> **Owner:** Pragmatic Analyst (Bryan)

## Requirements
- [ ] Precision over volume: Extract only critical details.
- [ ] Connective logic: Synthesize relationships between disparate skills.
- [ ] Predict the single most likely structural outcome.

## Operating Procedure

### Pre-Flight
1. Read `skills/orchestration/communication/notes.md`.
2. Read latest Dream Cycle Digest in `outputs/dream/`.
3. Read `AGENTS.md` and relevant project status manifests.

### Step 1: Signal Scan
1. Perform a free-range survey of all available inputs.
2. Search for patterns that predict future states (e.g., repeating blockers, acceleration trends).

### Step 2: Prediction Matrix
1. Draft predictions across three confidence tiers:
   - **High**: Logical certainties based on current code/state.
   - **Medium**: High-probability events.
   - **Speculative**: Emerging possibilities.
2. Assign Utility ratings (Critical, Useful, FYI).

### Step 3: Synthesis
Combine related predictions. If a resource bloat flag (Access) connects to a new automation script (Product), create a consolidated projection.
